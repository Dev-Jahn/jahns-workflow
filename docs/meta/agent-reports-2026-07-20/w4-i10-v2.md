## 구현한 계약

- fixture를 실제 `_packet_registry()`에서 만들고 `_build_packet(..., routing_note=...)` 다음 `_render_prompt(...)`를 호출한다.
- 양성은 goal/title, packet의 canonical declared scope와 현행 worker-facing generic scope/worktree 경계, acceptance, 자기 판정 금지와 별도 verifier 판정, `WAYSTONE_REPORT.yaml` stanza의 네 필드를 단언한다.
- TASK_BLOCK 전체 ordered line equality로 `status`, `milestone`, `round`, `anchor`, `routing_note` 다섯 known-debt와 dependency의 `done` 상태를 exact pin한다. 제거와 TASK_BLOCK 내부 추가가 모두 RED다.
- `tasks.yaml`, `ROADMAP`, `PROGRESS`, `.waystone/`, `round close`, `exposure`, `overlay`를 I-10/Addendum 근거 주석이 붙은 대표 표면으로 case-insensitive 부재 단언한다.
- TASK_BLOCK 밖 추가 또는 template-origin 추가도 빠져나가지 않도록 현재 template SHA-256과 정확히 세 치환(`TASK_BLOCK`, `ACCEPTANCE`, `BASE_SHA`)으로 만든 전체 rendered prompt를 독립 oracle로 고정한다.

현재 adapter는 concrete `declared_scope` 경로를 prompt에 직접 출력하지 않는다. acceptance의 current-green 조건과 Addendum의 현행 동작 고정에 따라 테스트는 packet의 canonical scope 보존과 template의 실제 generic scope/worktree 경계를 각각 정직하게 단언하며, `notes`에 scope 값을 복제해 투영된 것처럼 가장하지 않는다.
